## 为什么要拆开？

原来把数组、字符串、结构体、地址、栈和堆都放在同一篇里，读起来会很挤。它们彼此相关，但学习难度不同：

- 数组和字符串更偏基础数据组织。
- 字符数组涉及 C 风格字符串和底层细节。
- 指针、引用、栈、堆涉及内存模型，应该放慢讲。

拆分后，每篇文章只解决一个主题，示例和练习也能更具体。

## 本节小结

- 复合类型的核心是“把多个数据组织起来”。
- 内存模型的核心是“数据放在哪里、什么时候释放”。
- 遇到具体问题时，优先进入对应细分章节学习。
