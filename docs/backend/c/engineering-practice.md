# C 工程实践

C 语言入门不只是在单个文件里写题。随着程序变大，需要逐步建立工程习惯：拆分文件、命名清晰、检查错误、调试循环和内存问题。

## 文件组织

小程序可以放在一个 `.c` 文件里。稍大一点时，可以拆成头文件和实现文件。

```text
project/
  main.c
  math_utils.h
  math_utils.c
```

头文件放声明：

```c
// math_utils.h
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

int max(int a, int b);

#endif
```

实现文件放定义：

```c
// math_utils.c
#include "math_utils.h"

int max(int a, int b) {
    return a > b ? a : b;
}
```

## 编译多个文件

```bash
gcc main.c math_utils.c -o app
```

如果提示“未定义引用”，通常是声明存在，但实现文件没有参与编译。

## 命名习惯

- 变量名表达含义：`score` 比 `s` 更清楚。
- 函数名表达动作：`calculate_sum`、`print_array`。
- 常量可以用大写：`MAX_SIZE`。

## 调试习惯

1. 先确认输入是否符合预期。
2. 打印关键变量的中间值。
3. 缩小问题范围，只保留最小复现代码。
4. 对数组和字符串优先检查下标和容量。

## 练习方式

翁恺课程里的编程训练很适合按下面节奏做：

1. 先独立写出第一版。
2. 编译，读错误信息。
3. 用一组正常数据测试。
4. 用边界数据测试，例如 0、1、负数、最大长度。
5. 复盘错误类型，整理到自己的错题笔记。

## 小结

- C 工程习惯从多文件组织开始。
- 头文件声明接口，`.c` 文件实现逻辑。
- 编译错误要读，运行错误要打印中间状态。
- 数组、指针、字符串问题优先检查边界。
